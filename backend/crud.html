CRUD

What is CRUD?

This is what Wikipedia says:

In computer programming, create, read, update and delete (as an acronym CRUD or possibly a Backronym) (Sometimes called SCRUD with an "S" for Search) are the four basic functions of persistent storage

What do we need now is to analyze how do we want to structure the routes. But... what does de routes have to do with the CRUD? The answer is... EVERYTHING.

I am going to explain it very simple:

  • Our application has users.
    • Users have bags
      • Bags have items and trips

It means that a bag only will be created/updated/modified/deleted inside a certain user, and items and trips inside a certain bag as well. In a rails application, it will look like that:

/users/:user_id/bags/:bag_id/items/:id

So, we will have something like:

/users/1/bags/3/items/23

And what is that? It means that the user with id number is inside his bag with id number 3 watchin his item with id number 23. Ok, it seems easy to understand but hard to build, isn't it?

  1. Change the routes

We are going to say to our application that we want to build the routes in the way I explained before. We have to go to the file "routes.rb" and insert the following line:

resources :users do
      resources :bags do
          resources :items
          resources :trips
      end
  end

This is an example for this application. Depending on your application design, it could change. But if you see, it is very intuitive.

It is automatically going to create all the routes we need for the CRUD operations. Don't be afraid about how to create these operations, we will see it later. First, we need to understand how the routes work.

Go to the terminal and type:

$ rake routes

It is going to show you all the available routes the application recognizes and the future CRUD operations they manage (these operations are in the right colunm). Check it:

[IMAGE]

Check the right column (Controller#Action). Action is the CRUD operation it manages. Let's analyze "items#create".

It means that the items controller (in the folder: controllers/items_controler.rb) should have a method called "create", which creates a new item, and it does it in the route "/users/:user_id/bags/:bag_id/items(.:format)"

NOTE that a route is not a view. Views are shown under a certain path, that's right. but remember it is not the same at all.
