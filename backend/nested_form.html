Nested Form

In our example, we are going to illustrate what is a nested form and how it works. I thought it could be important to do it before other easier forms, because you will probably need them for your future project.

In our case, we have one "trip" that belongs to a "bag". We have to declare "trip" as a nested attribute of bag. It means that a trip can not be created without a bag, and a bag has to have a trip. We are creating a "one to one" relationship.

In the model

This is how the bag model should look like now

#app/models/bag.rb
class Bag < ActiveRecord::Base
    belongs_to :user
    has_many :items
    has_one :trip
    accepts_nested_attributes_for :trip

    after_initialize do
        self.trip ||= self.build_trip()
      end

    validates :bagname, presence: true,length: { maximum: 30 }, format: { with: /\A[a-zA-Z0-9' ']*\z/ }
end

The important line here is the "after initialize", which will let you have access to the "trip" attribute of bag. Let's see now the form again, now rewrited to be a real "nested form"

In the view

#app/views/users/show.html.erb
<%= form_for @bag, url: {controller: "bags", action: "create", user_id: @user.id} do |f| %>
        <%=f.label :bagname  %>
        <%=f.text_field :bagname  %>
        <br/>
        <%= f.label :description %>
        <%= f.text_area :description %>
        <br/>
        <%= f.fields_for :trip do |trip_form| %>
            <%=trip_form.label :city  %>
            <%=trip_form.text_field :city  %>
            <br/>
            <%=trip_form.label :country  %>
            <%=trip_form.text_field :country  %>
            <br/>
            <%= trip_form.datetime_select(:from_date, default: Time.now + 1.days ) %>
            <br/>
            <%= trip_form.datetime_select(:to_date, default: Time.now + 2.days ) %>
            <br/>
        <% end  %>
        <%= f.submit "Save" %>
<% end %>

Now, you can see that we access to the trip attribute like that: ":trip"

Last step:

In the controller

#app/controllers/bags_controller.rb
def create
        user = User.find(params[:user_id])
        @bag = user.bags.new bag_params
        if params[:cancel]
            redirect_to action: 'show', controller: 'users', id: user.id

        elsif @bag.save
            @bag.reload.trip
            flash[:notice] = 'Bag was created successfully'
            redirect_to action: 'show', controller: 'users', id: user.id
        else
            flash[:alert]="Bag couldn't be created"
            redirect_to action: 'show', controller: 'users', id: user.id
        end
    end

    def new
        @bag = Bag.new
        @bag.trip.build
    end

    private
    def bag_params
        params.require(:bag).permit(:bagname, :description, :user_id, trip_attributes: [:id,:city,:country, :from_date, :to_date])
end

This is the most important part.

In the "new" method, we have to say that we "build" the trip attribute for the bag.

Then, in the "create" method, we have to:

  1. Create the bag, from the user object
  2. Once we save it, reload the trip from the bag object.
  3. When we create it, we have to indicate in the permited params that we are sending the trip attributes and which ones are we sending.

Have you realized that trip is allways in singular? This is because it is only "one" object, a bag is only going to has a trip, and a trip is only to belong to a bag. Otherwise, the name would have been in plural.
